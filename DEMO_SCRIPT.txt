[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture flow.
First, every query enters through the START node and is immediately scanned by our Security Checkpoint. 
If it flags prompt injection or a surgical procedure lacking owner consent, it routes to our Security Event Handler to block execution.
Safe queries pass to our Pet Care Orchestrator, which is the central LLM brain.
The Orchestrator delegates specialized tasks to our three sub-agents: the Appointment Agent, the Medication Agent, or the Routine Agent.
Whenever a new appointment is proposed, the flow moves to the HITL Approval Gate, which pauses execution to wait for user confirmation before calling our local Model Context Protocol, or MCP, server to finalize the booking.
Finally, the results are formatted and sent to the Web UI via the Final Output node.
